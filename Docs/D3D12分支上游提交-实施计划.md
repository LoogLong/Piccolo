## 阶段一 · 移除 lganim 及全部残留

lganim 是一套独立的 Motion Matching / LMM 原型，`AnimationComponent` 被**整体改写**为依赖它。移除需要：删目录 + 还原集成点 + 删专属资产 + 还原专属数学 + 还原 demo 资产。

### 1.1 删除 lganim 目录（11 个文件）
删除整个 `engine/source/runtime/function/animation/lganim/`：
`anim_instance.{h,cpp}`、`mm_instance.{h,cpp}`、`single_instance.{h,cpp}`、`database.h`、`neural_network.h`、`spring.h`、`array.h`、`common.h`。

### 1.2 还原被 lganim 侵入的集成点（还原到上游 `f7a2751`）
以下文件的改动**全部**为 lganim 服务，直接还原到上游版本：

| 文件 | 处理 | 说明 |
| --- | --- | --- |
| `.../component/animation/animation_component.h` | 还原上游 | 去掉 `CAnimInstanceBase* m_anim_instance`、`GetComponentResult/GetParentInfo/HasRootMotion/GetRootMotion/getInstance`，恢复 `Skeleton m_skeleton`（值语义） |
| `.../component/animation/animation_component.cpp` | 还原上游 | 恢复 `postLoadResource/tick/getResult/getSkeleton` 原实现；去掉帧计数去重与 MM 实例 |
| `.../component/motor/motor_component.cpp` | 还原上游 | 去掉 root-motion 分支与对 `AnimationComponent::HasRootMotion/GetRootMotion` 的调用 |
| `.../framework/level/level_debugger.cpp` | 还原上游 | 该 diff 全属 MM 调试可视化（`drawMotionMatching`、`GetComponentResult`），整体还原（同时恢复 `g_is_editor_mode` 提前返回） |
| `.../framework/level/level_debugger.h` | 还原上游 | 去掉 `drawMotionMatching` 声明与相关 include |
| `.../framework/world/world_manager.h` | 还原上游 | 去掉 `getFrameCount/setFrameCount/m_frame_count`（仅 MM 一帧一更新用） |
| `engine/source/runtime/engine.cpp` | 还原该行 | 去掉 `m_world_manager->setFrameCount(getFrameCount())` |
| `engine/source/runtime/engine.h` | 还原该行 | 去掉 `getFrameCount()`（若无其它引用） |

> 已核实：**渲染/GPU Skinning 侧不消费** `AnimationComponent::GetComponentResult()` 等 MM 接口，仅使用上游也存在的 `getResult()`（`AnimationResult`）。因此还原不会破坏 D3D12 渲染与 skinning。

### 1.3 删除 lganim 专属二进制资产（合计 ~72 MB）
删除 `engine/asset/animation/` 下：`database.bin`(64MB)、`my_latent.bin`、`my_decompressor.bin`、`character.bin`。这些仅被 `mm_instance.cpp` 硬编码路径加载。

## 阶段三 · 其余问题整改方案（分批提交）

> 目标：让**渲染后端** PR 达到上游质量。按“确认 bug → 解耦 → 工程化 → 清理”分批，可各自独立成 commit/PR。

### 批次 A：确认的 Bug（优先，低风险高收益）
- **A1** `particle_pass.cpp:1298-1300`：描述符集分配失败时 `LOG_INFO("...done")` 改为 `throw std::runtime_error(...)`（与上一处对齐）；并修正 `~992` 成功路径的误导性异常文案。
- **A2** `render_scene.cpp:~242-244`：删除/修正 `skipped_skinned` 陈旧日志（该变量从不自增，且 skinned mesh 已纳入）。
- **A3** `render_resource.cpp:~41-43`：实现 `RenderResource::clear()`，释放 PT 缓冲、BLAS、skinned 输出，避免关闭/重载泄漏。
- **A4** 非 DXR/Vulkan 环境下 `RenderSceneMode=PathTracing` 静默回退光栅：启动时输出可见 `LOG_WARN`（请求模式 ≠ 生效模式）。

### 批次 B：RHI 解耦（高内聚低耦合）
- **B1** 以 RHI 能力查询替换 pass/pipeline/resource 层散落的 `getBackendType()==D3D12`（16 处）：新增如 `supportsRayTracing()`、`supportsPathTracing()`、`requiresPreSubmitDepthCopy()` 等能力接口。
- **B2** 将 `rhi_ray_tracing.h` 中的 `wchar_t*` export 名常量（`kPathTracing*Export`）移出通用 RHI 头，改放路径追踪 pass/shader 元数据；RHI 层使用中立的 `const char*` 或后端各自映射。
- **B3** 明确光追定位：实现 Vulkan 光追，或在接口/文档标注“光追当前仅 D3D12（实验性）”，并让 Vulkan 能力查询返回“不支持”，杜绝上层误用空实现。
- **B4** ImGui 后端初始化收敛为中立的 `RHIUiBackendInitInfo`，减少 `ui_pass.cpp` 对 `D3D12RHI/VulkanRHI` 的直接 `static_cast`。

### 批次 C：D3D12 后端工程化
- **C1** 拆分 `d3d12_rhi.cpp`（10520 行）为多个编译单元：`d3d12_resource`、`d3d12_format_barrier`、`d3d12_descriptor`、`d3d12_swapchain`、`d3d12_raytracing`、`d3d12_pipeline`。
- **C2** 删除或接线 `D3D12DescriptorHeapAllocator`（`d3d12_descriptor_heap.*`，当前未被使用）。
- **C3** 修复 `createBufferWithAllocation` 的 allocation 契约：D3D12 侧实现 allocation，或明确不支持并在 debug 断言。
- **C4** 补光追 pipeline/SBT 销毁路径（新增 `destroyRayTracingPipeline` 或在 `PathTracingPass` 析构清理）；停止在 `cmdTraceRays` 内改写共享 pipeline 的 `layout`。
- **C5** 命令录制失败路径（`cmdDraw/cmdBindDescriptorSets/cmdTraceRays` 等）至少加 debug assert，避免静默出错帧。
- **C6** 魔法数字（描述符堆 65536/2048、RTV/DSV 1024、RT payload/attributes 等）改为具名常量或可配置。

### 批次 D：清理（低风险）
- **D1** 移除残留调试日志：`particle_pass.cpp:~994/~1088/~1300` 无条件 `LOG_INFO`。
- **D2** 删除死代码/未用字段：`path_tracing_pass` 的 `write_count/p_writes`、`m_descriptor_set_dirty` 等只写不读字段、`static_data_changed` 不可达分支。
- **D3** 清理 `build_windows.bat`（`//cmake` 非法注释、`del` 缺 `/Q`）与 `engine/shader/CMakeLists.txt`、`runtime/CMakeLists.txt` 中大段注释掉的旧逻辑。
- **D4** 评估 `main_camera_pass` 中 `setupPathTracingCompositeRenderPass()` 与主 pass 的重复，抽取公共 subpass 构建以降低漂移风险（较大，可延后）。

### 批次 E：性能（可延后，非阻断）
- TLAS 增量更新替代每帧全量重建；静态 BLAS 复用而非 destroy+recreate；GPU skinning 批量化描述符/dispatch。
